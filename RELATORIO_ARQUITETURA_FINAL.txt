════════════════════════════════════════════════════════════════════════════════
                    MÓDULOS ENTREGUES E RESPONSABILIDADES
════════════════════════════════════════════════════════════════════════════════

┌─ database_refactored.py (Schema Otimizado)
│  ├─ Usuários unificados com roles
│  ├─ Elimina cursos duplicados
│  ├─ Relacionamentos corretos (Curso -> Turma -> Matrícula -> Aluno)
│  ├─ Tabelas: Parcelas, Frequências, Notas, Observações
│  └─ Hierarquia: Admin (100) > Diretoria (90) > Pedagogo (60) > Comercial (40) > Professor (20) > Aluno (10)

┌─ controle_acesso.py (RBAC - Role Based Access Control)
│  ├─ PermissaoManager: verifica permissões granulares
│  ├─ ControladorAcesso: interface centralizada
│  ├─ Decorators: @requer_permissao, @requer_roles
│  ├─ Filtros automáticos: turmas_visiveis(), alunos_visiveis()
│  └─ Suporta wildcards e permissões compostas

┌─ logica_financeira.py (Gestão de Parcelas e Receita)
│  ├─ GeradorParcelas: cria automaticamente na matrícula
│  ├─ Calcula receita mensal (soma de pagos)
│  ├─ Calcula pendentes totais e por aluno
│  ├─ Registra pagamentos com rastreabilidade
│  ├─ Relatório de inadimplentes
│  └─ RangerFinanceiro: relatórios diários/mensais

┌─ logica_academica.py (Frequência, Notas, Observações)
│  ├─ GerenciadorFrequencia: registro de chamada
│  ├─ GerenciadorNotas: cálculo automático de média
│  ├─ GerenciadorObservacoes: histórico de comportamento
│  └─ Consolidação de histórico completo do aluno

👨‍🎓 ALUNO (nível 10)

   ├─ Ver observações sobre si

════════════════════════════════════════════════════════════════════════════════
                    FLUXOS IMPLEMENTADOS
════════════════════════════════════════════════════════════════════════════════

FLUXO 1: Matrícula com Parcelas Automáticas
   Comercial: Cria Aluno → Seleciona Turma → Define Parcelas
   Sistema: Cria Matrícula → Gera Parcelas Automáticas → Registra no DB
   Resultado: Parcelas aparecem no Financeiro imediatamente

FLUXO 2: Registro de Frequência
   Professor: Acessa Turma → Seleciona Data → Marca Presença
   Sistema: Salva com data/hora e professor_id
   Aluno: Vê frequência consolidada em seu perfil

FLUXO 3: Registro de Notas
   Professor: Insere 3 notas → Sistema calcula média automática
   Aluno: Vê notas e status (aprovado/recuperação/reprovado)

FLUXO 4: Pagamento de Parcelas
   Comercial: Marca parcela como Pago → Data preenchida automaticamente
   Dashboard: Receita do mês recalculada em tempo real
   Aluno: Vê extratos financeiros atualizados

FLUXO 5: Histórico Completo do Aluno
   Pedagogo/Diretoria: Acessa /aluno/<id>/perfil
   Sistema: Mostra frequência, notas, observações, financeiro consolidado
   Resultado: Visão 360° do aluno

                    ARQUITETURA TÉCNICA
════════════════════════════════════════════════════════════════════════════════

PADRÃO DE DESIGN: MVC + Service Layer

┌─ Models (database_refactored.py)
│  └─ Esquema normalizado, sem redundância

┌─ Services (controle_acesso.py, logica_financeira.py, logica_academica.py)
│  ├─ PermissaoManager: regras de acesso
│  ├─ GeradorParcelas: lógica financeira
│  ├─ Gerenciadores Acadêmicos: frequência, notas, observações

┌─ Controllers (rotas em app.py)
│  ├─ @requer_autenticacao
│  ├─ @PermissaoManager.requer_permissao(...)
│  ├─ Delegam para services
│  └─ Retornam templates

SEGURANÇA:
  ✅ Decorators protegem cada rota
  ✅ Validação de permissão em cada ação
  ✅ Auditoria em audit_log
  ✅ Isolamento de dados por role

PERFORMANCE:
  ✅ Queries otimizadas com índices
  ✅ Filtros aplicados no SQL, não em Python
  ✅ Cache de role em session
  ✅ Bulk inserts para parcelas
